# dida365 Task Sync Skill - 使用示例

本文档提供实际使用场景和示例对话。

## 场景 1: 基本同步

**用户请求:**
```
"帮我把 dida365 今天工作列表的任务同步到 ~/Documents/tasks.md"
```

**WorkBuddy 处理步骤:**
1. 加载 dida365-sync skill
2. 加载 Dida365 CLI skill
3. 从 dida365 获取今日工作列表任务
4. 读取 ~/Documents/tasks.md
5. 查找或创建今日日期标题
6. 更新或添加任务
7. 写入更新后的文档
8. 报告同步结果

**示例输出:**
```
已成功同步 5 个任务到 ~/Documents/tasks.md

变更摘要:
- 更新了 3 个现有任务的完成状态
- 添加了 2 个新任务
- 保留了 1 个手动添加的任务
```

## 场景 2: 指定日期同步

**用户请求:**
```
"同步 dida365 2026-03-25 的任务到我的待办文档"
```

**处理流程:**
- 从 dida365 获取 2026-03-25 的任务
- 查找目标文档中的日期标题 "2026年3月25日（周三）"
- 更新或创建该日期的任务列表

## 场景 3: 首次创建文档

**用户请求:**
```
"创建一个新的任务文档 ~/Documents/daily_tasks.md，并同步 dida365 今天的任务"
```

**处理流程:**
- 创建新文档 ~/Documents/daily_tasks.md
- 添加今日日期标题和任务
- 格式化输出

**生成的内容:**
```markdown
# 2026年3月26日（周四）
- [x] 安装新版iso
  - [x] 本地
  - [x] 151机器
- [ ] 继续集成测试
```

## 场景 4: 同步多个项目

**用户请求:**
```
"分别把 dida365 的项目和开发列表同步到不同的文档"
```

**处理流程:**
- 分别查询不同列表的任务
- 分别同步到对应的文档
  - ~/Documents/work_tasks.md (工作列表)
  - ~/Documents/dev_tasks.md (开发列表)

## 场景 5: 查看变更预览

**用户请求:**
```
"先预览一下今天的任务同步变更，不直接写入"
```

**处理流程:**
- 获取 dida365 任务
- 读取本地文档
- 比较差异
- 显示将要进行的变更
- 等待用户确认后再执行

**示例输出:**
```
📋 预览同步变更 (2026年3月26日):

将要添加的任务:
  - [ ] 代码审查
  - [ ] 更新文档

将要更新的任务:
  "安装新版iso" 状态: 完成 → 未完成

将要保留的任务 (本地手动添加):
  - [ ] 购买咖啡豆

确认执行同步吗? (y/n)
```

## 场景 6: 错误处理

### dida365 连接失败

**用户请求:**
```
"同步今天的任务"
```

**错误输出:**
```
❌ 同步失败: 无法连接到 dida365
请检查:
1. Dida365 CLI 是否已安装和配置
2. 网络连接是否正常
3. API Key 是否有效
```

### 文件写入失败

**用户请求:**
```
"同步到 ~/readonly/tasks.md"
```

**错误输出:**
```
❌ 写入文件失败: 权限不足
目标文件: ~/readonly/tasks.md
请检查文件权限或选择其他位置
```

## 场景 8: 增量更新

**用户请求:**
```
"每隔一小时自动同步一次今天的任务"
```

**处理流程:**
- 每次同步只更新变更的任务
- 不删除本地手动添加的任务
- 保持文档结构稳定

## 最佳实践

1. **定期同步**: 建议设置定时任务，每天开始工作前同步一次
2. **备份文档**: 重要文档建议定期备份
3. **查看变更**: 首次使用建议先预览变更
4. **手动任务**: 手动添加的任务会被保留，可用于补充 dida365 外的任务

## 常见问题

### Q: 为什么要使用这个 skill 而不是手动复制？

A: 这个技能提供自动化同步，节省时间，并且：
- 确保任务状态一致性
- 避免重复输入
- 支持批量处理
- 可以定时自动执行

### Q: 如果我在本地修改了任务，会被覆盖吗？

A: 不会。技能会：
- 保留本地手动添加的任务
- 只更新任务的完成状态
- 不会删除本地内容

### Q: 支持同步多个日期的任务吗？

A: 支持。可以指定任何日期，但一次只能同步一个日期。

### Q: 如何自定义日期格式？

A: 可以修改 `scripts/dida365_sync.py` 中的 `format_date_header()` 函数。
