# dida365 Working 项目同步技能

## 用途

专用于将滴答清单 **working 项目** 的每日任务同步到本地 Markdown 文档。自动区分待完成和已完成任务，保持一致的格式。

## 使用场景

- 每日同步 working 项目任务到工作笔记
- 查看今日任务完成状态
- 保持任务历史记录

## 固定配置

| 配置项 | 值 | 说明 |
|--------|-----|------|
| **项目** | working (ID: 1737466f9955964c71817638) | 固定不变 |
| **基础目录** | `D:\CodeAndNotes\records\02-TaskNotes\OS` | **必须已存在**，可修改 |
| **文件名格式** | `WorkNotes-YYYYMM-YYYYMM.md` | 双月制，自动计算 |
| **日期格式** | `# YYYY年M月D日（周X）` | 中文星期 |

### 文件名规则（双月制）

| 月份 | 文件名示例 |
|------|-----------|
| 1-2月 | `WorkNotes-202601-02.md` |
| 3-4月 | `WorkNotes-202603-04.md` |
| 5-6月 | `WorkNotes-202605-06.md` |
| ... | ... |

**格式说明**: `WorkNotes-年份起始月-结束月.md`（结束月只显示月份，不带年份）

### 迁移配置

如需更改基础目录，修改脚本中的 `BASE_DIR` 变量：

```python
# scripts/dida365_working_sync.py
BASE_DIR = r"D:\CodeAndNotes\records\02-TaskNotes\OS"  # 修改此处
```

## 输出格式

```markdown
# 2026年3月31日（周二）

- [ ] 任务A
  - [ ] 子任务1
- [ ] 任务B

- [x] 任务C
  - [x] 子任务2
```
### 输出格式补充说明
完成为：
- [x] 任务C
未完成为：
- [ ] 任务B

## 工作流程

1. **检查基础目录**
   - 确认 `BASE_DIR` 存在
   - 如不存在，提示用户修改配置

2. **检查目标文件**
   - 根据日期计算文件名（双月制）
   - 如文件不存在，提示用户创建或授权自动创建

3. **获取 working 项目今日待完成任务**
   - 调用 `dida365 project show 1737466f9955964c71817638`
   - 筛选 `dueDate` 为今日且 `status=0` 的任务

4. **获取 working 项目今日已完成任务**
   - 调用 `dida365 completed today`
   - 筛选 `projectId=1737466f9955964c71817638` 的任务

5. **生成 Markdown 内容**
   - 按状态分组：待完成 / 已完成
   - 包含子任务（缩进2空格）

6. **更新目标文件**
   - 读取现有文件
   - 定位或创建今日日期区块
   - 追加 Working 项目任务
   - 保存文件

## 任务状态映射

| dida365 | Markdown | 说明 |
|---------|----------|------|
| `status=0` | `- [ ]` | 待完成 |
| `status=2` | `- [x]` | 已完成 |
| `items` 子任务 | 缩进2空格 | 子任务列表 |

## 错误处理

- 如果基础目录不存在，提示用户修改 `BASE_DIR` 配置
- 如果目标文件不存在，交互式询问：手动创建 / 自动创建 / 退出
- 如果 dida365 连接失败，报告错误并停止
- 如果文件写入失败，保留备份并报告

## 使用方式

加载此技能后，直接执行同步即可：

```
同步今日 working 项目任务
```

或调用脚本：

```bash
# 同步今天
python scripts/dida365_working_sync.py

# 同步指定日期
python scripts/dida365_working_sync.py 2026-03-31
```

## 注意事项

1. **基础目录必须已存在**：脚本不会自动创建目录
2. **文件名自动计算**：根据日期自动匹配双月文件
3. **交互式确认**：文件不存在时会询问用户处理方式
